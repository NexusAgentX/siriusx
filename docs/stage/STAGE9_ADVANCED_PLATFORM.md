# Stage 9：平台生态 MVP

> 一个完整的平台化产品：支持企业权限、外部集成、高级执行策略和工作流能力。

---

## 这层产品是什么

Stage 9 是平台扩展层。前面阶段证明产品能用、敢用、能运营；这一层让它进入企业流程和生态集成。

Stage 9 不是一次性大包，每个能力都应该单独做成小 MVP。

---

## 可选 MVP 子层

### 9A：企业权限 MVP

- RBAC。
- tenant admin console。
- 团队空间。
- 审计导出。
- 组织级策略。

### 9B：集成 MVP

- API token。
- Webhook。
- 外部系统回调。
- Git provider 集成。
- CI/CD 集成。

### 9C：执行优化 MVP

- per-task run queue。
- sandbox pool。
- 节点亲和性。
- S3 增量上传。
- 本地 blob cache。

### 9D：工作流 MVP

- 多 Agent 协作。
- 人工审批节点。
- 定时任务。
- 批量任务。
- workflow template。

---

## 明确不做

- 不把所有高级能力一次性作为一个大版本交付。
- 不为了平台化破坏前面稳定的核心任务链路。
- 不允许自定义工具绕过权限和审计。

---

## 完成标准

每个 Stage 9 子能力都必须满足：

- 有明确用户场景。
- 能独立启用或关闭。
- 有权限模型。
- 有审计路径。
- 有回滚策略。
- 不破坏 Stage 0-8 的核心体验。

---

## 这一层的 MVP 价值

SiriusX 从单一产品变成平台：企业可以把它接入权限体系、业务系统和自动化流程。
