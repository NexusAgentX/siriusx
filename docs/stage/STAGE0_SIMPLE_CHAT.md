# Stage 0：最小聊天 MVP

> 一个完整可用的最小产品：用户打开网页，输入问题，AI 流式回答。

---

## 这层产品是什么

Stage 0 是 SiriusX 的最小最小版本。它不是技术骨架，而是一个能给用户价值的聊天产品。

用户价值很简单：用户可以把它当成一个私有 AI 聊天入口，问问题、获得回答、继续追问。

---

## 核心用户路径

```text
打开首页
  -> 看到一个聊天窗口
  -> 输入问题
  -> AI 流式回复
  -> 用户继续追问
  -> 页面保留本次对话
```

---

## 必须有

- 一个可访问的 Web 页面。
- 一个输入框。
- 一个发送按钮。
- 一个停止生成按钮。
- 模型流式回复。
- 当前浏览器内的对话历史。
- 基础错误提示。

Stage 0 可以不登录，可以不建数据库。为了让用户刷新后不丢当前体验，可以先用浏览器 localStorage 保存最近会话。

---

## 最小 API

```text
POST /api/chat
  input:  messages[]
  output: streaming assistant response
```

如果实现更简单，也可以先不设计完整 REST 资源，只保留一个能跑通聊天的 endpoint。

---

## 明确不做

- 不做登录。
- 不做用户系统。
- 不做任务列表。
- 不做工具调用。
- 不做文件工作区。
- 不做数据库持久化。
- 不做队列。
- 不做沙箱。
- 不做多 Agent。
- 不做历史管理页面。

---

## 完成标准

- 一个用户能连续聊 20 轮。
- 回复是流式出现的。
- 停止生成可用。
- 模型报错时页面不崩溃。
- 刷新页面后能保留最近一次会话，或至少明确提示当前版本只保存当前页面会话。
- 只需要一个前端服务和一个后端接口就能运行。

---

## 这一层的 MVP 价值

用户可以立即把 SiriusX 当成一个可用的 AI 对话入口。即使后面什么都不做，Stage 0 也不是废品。
