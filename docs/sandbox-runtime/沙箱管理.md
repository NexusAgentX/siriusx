# Sandbox 管理

> 阶段 0-2 采用 Stateless Sandbox 策略，无节点亲和性
> 本文件属于 `siriusx-sandbox-runtime`，不属于可信控制面

---

## 核心策略

**🚨 P1 修正：完全 Stateless，删除 sandbox_leases 表**

**每个 run 独立创建全新 sandbox，run 结束后立即销毁。**

控制面只发来短期 `RunSpec` 和 capability token；真正的 sandbox 生命周期、shell 执行和销毁都在沙箱运行时内部完成。

### 为什么 Stateless？

- **简化架构**：无需节点亲和性调度
- **故障恢复简单**：Control Worker 崩溃后任意 Control Worker 可接手（无 sandbox 状态依赖）
- **性能开销可接受**：Sandbox 创建 2-5 秒，单 run 平均 5-10 分钟，开销 < 2%

### 与 Lease 模型对比

| 特性 | Stateless（阶段 0-2） | Lease（未来扩展） |
|---|---|---|
| Sandbox 生命周期 | 随 run 创建/销毁 | 跨 run 复用 |
| 节点亲和性 | 不需要 | 需要 |
| `sandbox_leases` 表 | ❌ 不需要 | ✅ 需要 |
| 故障恢复复杂度 | 低 | 高 |
| 适用场景 | 早期生产 | 优化后（如发现开销 > 5%）|

## SandboxProvider 接口

> 仅限沙箱运行时内部使用。控制面对外只使用 `SandboxRuntimeClient`。

```typescript
interface SandboxProvider {
  // 创建短生命周期 sandbox
  create(input: {
    taskId: string;
    runId: string;
    workspaceDir: string; // 已 materialized 的本地目录
    baseImage: string;    // 如 siriusx/agent-sandbox:latest
    env?: Record<string, string>;
  }): Promise<Sandbox>;

  // 懒创建：如果不存在则创建，存在则返回现有 sandbox
  ensure(input: {
    taskId: string;
    runId: string;
    workspaceDir: string;
    baseImage: string;
    env?: Record<string, string>;
  }): Promise<Sandbox>;

  // 通过 runId 获取 sandbox（如果存在）
  getByRunId(runId: string): Promise<Sandbox | null>;

  // 执行命令
  exec(input: {
    sandboxId: string;
    command: string;
    cwd?: string;
    timeoutMs?: number;
    signal?: AbortSignal;
    onData?: (data: Buffer) => void;
  }): Promise<SandboxExecResult>;

  // 销毁 sandbox
  destroy(sandboxId: string): Promise<void>;

  // 清理 orphan 容器（沙箱运行时或宿主崩溃后遗留的容器）
  sweep(input: { maxAgeMinutes: number }): Promise<void>;
}

interface Sandbox {
  id: string;           // Docker container ID
  taskId: string;
  runId: string;
  provider: "local-docker";
  createdAt: number;
}

interface SandboxExecResult {
  exitCode: number;
  stdout: string;
  stderr: string;
  durationMs: number;
}
```

**关键方法说明**：
- `create()`：直接创建新 sandbox
- `ensure()`：懒创建策略，首次调用时创建，后续调用返回现有 sandbox
- `getByRunId()`：查询当前 run 是否已创建 sandbox（用于清理）

## 沙箱运行时执行流程

```typescript
async function executeRun(runSpec: RunSpec) {
  let sandbox: Sandbox | null = null;
  
  try {
    // 1. Materialize workspace 到本地
    const workspaceDir = await workspaceProvider.materialize({
      workspaceManifestRef: runSpec.workspaceManifestRef,
      taskId: runSpec.taskId,
      runId: runSpec.runId,
    });
    
    // 2. 不预判，不预创建 sandbox
    // sandbox 采用"懒创建"策略：首次 shell 工具调用时才创建
    
    // 3. 执行 Pi run（sandboxProvider 作为参数传入）
    await piRunner.run({
      ...runSpec,
      workspaceDir,
      sandboxProvider, // 传入 provider，由 Pi runner 在首次需要时调用 ensure()
    });
    
    // 4. 生成 workspace commit proposal，回传控制面校验和提交
    const proposal = await workspaceProvider.buildCommitProposal({ ... });
    await eventStream.emit({
      type: 'workspace_commit_proposal',
      payload: proposal,
    });
    
  } finally {
    // 5. run 结束后立即销毁 sandbox（如果创建过）
    // sandbox 引用由 Pi runner 管理，通过 runId 查询是否存在
    const sandbox = await sandboxProvider.getByRunId(runSpec.runId);
    if (sandbox) {
      await sandboxProvider.destroy(sandbox.id).catch(err => {
        logger.warn(`Failed to destroy sandbox ${sandbox.id}`, err);
      });
    }
  }
}
```

**🚨 懒创建策略说明**：
- 不使用 `checkNeedsSandbox()` 预判（实际是否需要 shell 在 Pi tool call 时才知道）
- `SandboxProvider.ensure(runId)` 方法：如果不存在则创建，存在则返回现有 sandbox
- Pi runner 在首次执行 bash/shell 工具时调用 `ensure()`
- 避免为纯文本对话创建不必要的 sandbox

## LocalDockerSandboxProvider 实现

```typescript
class LocalDockerSandboxProvider implements SandboxProvider {
  private docker = new Docker();
  
  async create(input: CreateSandboxInput): Promise<Sandbox> {
    const container = await this.docker.createContainer({
      Image: input.baseImage,
      Labels: {
        'siriusx.managed': 'true',
        'siriusx.taskId': input.taskId,
        'siriusx.runId': input.runId,
        'siriusx.createdAt': Date.now().toString(),
      },
      HostConfig: {
        Binds: [
          `${input.workspaceDir}:/workspace:rw`,
        ],
        NetworkMode: 'bridge',
        Memory: 2 * 1024 * 1024 * 1024, // 2GB
        CpuQuota: 100000, // 1 CPU
      },
      WorkingDir: '/workspace',
      Env: Object.entries(input.env || {}).map(([k, v]) => `${k}=${v}`),
    });
    
    await container.start();
    
    return {
      id: container.id,
      taskId: input.taskId,
      runId: input.runId,
      provider: 'local-docker',
      createdAt: Date.now(),
    };
  }
  
  async exec(input: ExecInput): Promise<SandboxExecResult> {
    const container = this.docker.getContainer(input.sandboxId);
    
    const exec = await container.exec({
      Cmd: ['sh', '-c', input.command],
      AttachStdout: true,
      AttachStderr: true,
      WorkingDir: input.cwd || '/workspace',
    });
    
    const startTime = Date.now();
    const stream = await exec.start();
    
    let stdout = '';
    let stderr = '';
    
    stream.on('data', (chunk: Buffer) => {
      const data = chunk.toString();
      if (input.onData) {
        input.onData(chunk);
      }
      stdout += data;
    });
    
    await new Promise((resolve, reject) => {
      stream.on('end', resolve);
      stream.on('error', reject);
      
      if (input.timeoutMs) {
        setTimeout(() => reject(new Error('Exec timeout')), input.timeoutMs);
      }
    });
    
    const inspect = await exec.inspect();
    
    return {
      exitCode: inspect.ExitCode || 0,
      stdout,
      stderr,
      durationMs: Date.now() - startTime,
    };
  }
  
  async destroy(sandboxId: string): Promise<void> {
    const container = this.docker.getContainer(sandboxId);
    await container.stop({ t: 5 }); // 5 秒超时
    await container.remove({ force: true });
  }
  
  async sweep(input: { maxAgeMinutes: number }): Promise<void> {
    // 扫描带有 siriusx.managed=true 标签的容器
    const containers = await this.docker.listContainers({
      all: true,
      filters: {
        label: ['siriusx.managed=true'],
      },
    });
    
    const maxAgeMs = input.maxAgeMinutes * 60 * 1000;
    const now = Date.now();
    
    for (const containerInfo of containers) {
      const createdAt = parseInt(
        containerInfo.Labels['siriusx.createdAt'] || '0'
      );
      
      if (now - createdAt > maxAgeMs) {
        logger.info(`Sweeping orphan container ${containerInfo.Id}`);
        await this.destroy(containerInfo.Id);
      }
    }
  }
}
```

## Orphan 容器清理

沙箱运行时或宿主崩溃后可能留下孤儿容器。通过定时任务清理：

```typescript
// 每小时运行一次
setInterval(async () => {
  await sandboxProvider.sweep({
    maxAgeMinutes: 30, // 清理 30 分钟前的容器
  });
}, 60 * 60 * 1000);
```

**清理策略**：
- 通过 Docker labels 识别 SiriusX 管理的容器
- 清理超过 30 分钟的容器（正常 run 5-10 分钟）
- 不依赖数据库（Stateless 策略）

## 工具路由

Pi SDK 的 bash/shell 工具必须路由到 `SandboxProvider`，不能直接在控制面 host 上执行。

**🚨 懒创建策略实现**：

```typescript
class PiSdkRunner {
  constructor(
    private sandboxProvider: SandboxProvider,
    private workspaceDir: string,
    private runId: string,
    private taskId: string,
  ) {}
  
  async run(input: RunInput) {
    const session = await this.createSession({
      ...input,
      // 配置工具路由
      toolOverrides: {
        bash: async (args) => {
          // 懒创建：首次 bash 调用时才创建 sandbox
          const sandbox = await this.sandboxProvider.ensure({
            taskId: this.taskId,
            runId: this.runId,
            workspaceDir: this.workspaceDir,
            baseImage: 'siriusx/sandbox:latest',
          });
          
          return await this.sandboxProvider.exec({
            sandboxId: sandbox.id,
            command: args.command,
            cwd: args.cwd,
            timeoutMs: 5 * 60 * 1000, // 5 分钟超时
          });
        },
        shell: async (args) => {
          // shell 工具同样懒创建
          const sandbox = await this.sandboxProvider.ensure({
            taskId: this.taskId,
            runId: this.runId,
            workspaceDir: this.workspaceDir,
            baseImage: 'siriusx/sandbox:latest',
          });
          
          return await this.sandboxProvider.exec({
            sandboxId: sandbox.id,
            command: args.command,
            cwd: args.cwd,
            timeoutMs: 5 * 60 * 1000,
          });
        },
      },
    });
    
    return await session.run();
  }
}
```

**关键改进**：
- 不预判，不预创建 sandbox
- 首次 bash/shell 调用时通过 `ensure()` 懒创建
- 后续同一 run 的 bash/shell 调用复用已创建的 sandbox
- 纯文本对话不会创建 sandbox（节省资源）

## 性能分析

### 开销测算

| 指标 | 时间 |
|---|---|
| Sandbox 创建 | 2-5 秒 |
| Sandbox 销毁 | 1-2 秒 |
| 单 run 平均执行 | 5-10 分钟 |
| **总开销占比** | **< 2%** |

### 优化方向

**当前阶段**（阶段 0-2）：
- 优化 base image（预装常用依赖）
- 使用 Docker layer cache
- 并行创建多个 sandbox

**未来扩展**（阶段 3+）：
- 如果发现开销 > 5%，引入 task-scoped sandbox pool
- 这时才需要 `sandbox_leases` 表和节点亲和性调度

## 未来扩展：Sandbox Pool（阶段 3+）

**仅供参考，阶段 0-2 不实现**

如果 Stateless 策略的开销过大，可引入 task-scoped sandbox pool：

```typescript
// 一个 task 复用，多个 run 共享
interface SandboxLease {
  taskId: string;
  sandboxId: string;
  ownerNodeId: string;
  lastUsedAt: number;
  expiresAt: number;
}

// 需要节点亲和性调度
if task has ready sandbox on node A:
  prefer node A for next run
else:
  any worker can claim and create new sandbox
```

**这时才需要**：
- `sandbox_leases` 表
- 节点亲和性调度
- Lease heartbeat 机制
