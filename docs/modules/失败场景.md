# 失败场景

## Control Worker 崩溃

当 Control Worker 进程异常退出时，系统通过 lease 机制自动恢复：

```text
run lease expires
  -> another worker reclaims
  -> reconstruct context from durable state
  -> retry or mark failed based on attempt policy
```

## API 节点重启

API 节点重启后，客户端通过重新连接恢复会话：

```text
client reconnects
  -> reads Task snapshot
  -> subscribes latest active run
  -> backfills events by sequence
```

## Sandbox 创建失败

**🚨 阶段 0-2 Stateless Sandbox 策略**

Sandbox 创建失败时，run 标记为可重试错误：

```text
sandbox create timeout or failure
  -> mark run as retryable error
  -> worker releases lock
  -> queue delayed retry (exponential backoff)
  -> after max attempts mark run failed
```

## Workspace materialization 失败

Workspace 准备阶段失败时，系统支持延迟重试：

```text
run remains retryable
  -> worker releases lock
  -> queue delayed retry
  -> after max attempts mark run failed
```

## Pi auth / model 配置失败

认证或模型配置错误属于确定性错误，不进行重试：

```text
worker receives deterministic auth/config error
  -> mark run failed without retry
  -> persist operator-visible error code
  -> keep task open for config fix or user retry
```

## 事件持久化失败

**🚨 阶段 0-2 简化策略：不实现 S3 备份**

事件持久化失败时，Worker 通过本地 WAL 和重试机制尽力而为：

```text
ResultBus may have delivered realtime events
  -> Control Worker writes to local WAL (best-effort, 同进程内防丢失)
  -> appendEvents retry same batch (3 times, 统一退避: 5s, 30s, 120s；对齐 STAGE0_SPEC §6.4)
  -> if durable write still fails:
     - mark run completed_with_warnings
     - log warning with event count
     - Control Worker 崩溃后，内存缓冲中未持久化的事件已丢失
     - 新 Control Worker 只能从 Postgres 恢复已持久化的事件
```

**关键限制**：
- 阶段 0-2 **不保证事件不丢失**，只能尽力而为
- 前端无法从 S3 补齐缺失事件（未实现）
- 未来扩展（阶段 3+）才实现 S3 事件备份和恢复

## Workspace revision 冲突

当多个 run 并发修改 workspace 时，通过乐观锁检测冲突：

```text
run-002 tries to commit workspace
  -> checks base_workspace_revision against task.latest_workspace_revision
  -> another run already advanced the revision
  -> optimistic lock fails (UPDATE returns 0 rows)
  -> mark run failed with workspace_conflict_detected_please_retry
  -> user re-sends prompt, which will use the latest revision
```
