# 错误码规范

> 统一的错误处理和用户提示规范

---

## 错误响应格式

### 标准格式

```json
{
  "statusCode": 400,
  "code": "VALIDATION_ERROR",
  "message": "Title is required",
  "errors": [
    {
      "field": "title",
      "message": "Title must be between 1 and 200 characters"
    }
  ],
  "timestamp": 1718100000000,
  "path": "/api/task"
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `statusCode` | number | HTTP 状态码 |
| `code` | string | 业务错误码（大写下划线） |
| `message` | string | 用户友好的错误提示 |
| `errors` | array | 详细错误列表（可选，用于表单验证） |
| `timestamp` | number | 错误发生时间 |
| `path` | string | 请求路径 |

---

## HTTP 状态码使用规范

| 状态码 | 使用场景 |
|---|---|
| 200 | 成功（GET、PATCH） |
| 201 | 创建成功（POST） |
| 204 | 成功且无返回内容（DELETE、某些 POST） |
| 400 | 请求参数错误 |
| 401 | 未认证（需要登录） |
| 403 | 已认证但无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突（如重复创建） |
| 422 | 语义错误（如业务规则不满足） |
| 429 | 请求过多（Rate Limiting） |
| 500 | 服务器内部错误 |
| 503 | 服务暂时不可用 |

## 错误码分类

### Auth 类（AUTH_*）

| 错误码 | HTTP | 说明 | 用户提示 |
|---|---|---|---|
| `AUTH_REQUIRED` | 401 | 未登录 | 请先登录 |
| `AUTH_INVALID_TOKEN` | 401 | Token 无效 | 登录已过期，请重新登录 |
| `AUTH_SESSION_EXPIRED` | 401 | Session 过期 | 会话已过期，请重新登录 |
| `CSRF_TOKEN_MISSING` | 403 | CSRF token 缺失 | 请求缺少 CSRF token，请刷新页面 |
| `CSRF_TOKEN_INVALID` | 403 | CSRF token 无效 | CSRF token 无效，请刷新页面 |
| `INVALID_CREDENTIALS` | 401 | 凭据错误 | 邮箱或密码错误 |
| `EMAIL_ALREADY_EXISTS` | 409 | 邮箱已存在 | 该邮箱已被注册 |

## 前端错误处理

### 通用错误处理器

```typescript
async function handleApiError(response: Response) {
  const error = await response.json();
  
  // 根据错误码显示友好提示
  const userMessage = ERROR_MESSAGES[error.code] || error.message;
  
  // 显示 Toast 或 Modal
  toast.error(userMessage);
  
  // 特殊错误处理
  switch (error.code) {
    case 'AUTH_REQUIRED':
    case 'AUTH_SESSION_EXPIRED':
      // 跳转到登录页
      router.push('/login');
      break;
      
    case 'ACCESS_DENIED':
      // 跳转到无权限页
      router.push('/403');
      break;
      
    case 'WORKSPACE_CONFLICT':
      // 提示用户重试
      if (confirm('工作区已被修改，是否重试？')) {
        retryRequest();
      }
      break;
  }
  
  return error;
}
```

### React Hook 示例

```typescript
function useApiRequest<T>(
  fetcher: () => Promise<Response>
): {
  data: T | null;
  error: ApiError | null;
  loading: boolean;
  refetch: () => void;
} {
  const [data, setData] = useState<T | null>(null);
  const [error, setError] = useState<ApiError | null>(null);
  const [loading, setLoading] = useState(false);
  
  const execute = async () => {
    setLoading(true);
    setError(null);
    
    try {
      const response = await fetcher();
      
      if (!response.ok) {
        const error = await handleApiError(response);
        setError(error);
        return;
      }
      
      const result = await response.json();
      setData(result);
    } catch (err) {
      setError({
        code: 'NETWORK_ERROR',
        message: '网络连接失败',
      });
    } finally {
      setLoading(false);
    }
  };
  
  useEffect(() => {
    execute();
  }, []);
  
  return { data, error, loading, refetch: execute };
}
```

## 后端错误抛出

### NestJS 异常类

```typescript
import { HttpException, HttpStatus } from '@nestjs/common';

export class ApiException extends HttpException {
  constructor(
    code: string,
    message: string,
    statusCode: number = HttpStatus.BAD_REQUEST,
    errors?: Array<{ field: string; message: string }>
  ) {
    super(
      {
        statusCode,
        code,
        message,
        errors,
        timestamp: Date.now(),
      },
      statusCode
    );
  }
}

// 使用示例
throw new ApiException(
  'TASK_NOT_FOUND',
  'Task not found',
  HttpStatus.NOT_FOUND
);
```

### 快捷工厂方法

```typescript
export class ErrorFactory {
  static authRequired() {
    return new ApiException(
      'AUTH_REQUIRED',
      'Please login to continue',
      HttpStatus.UNAUTHORIZED
    );
  }
  
  static accessDenied() {
    return new ApiException(
      'ACCESS_DENIED',
      'You do not have permission to access this resource',
      HttpStatus.FORBIDDEN
    );
  }
  
  static taskNotFound(taskId: string) {
    return new ApiException(
      'TASK_NOT_FOUND',
      `Task ${taskId} not found`,
      HttpStatus.NOT_FOUND
    );
  }
  
  static runConflict() {
    return new ApiException(
      'RUN_CONFLICT',
      'Another run is already in progress',
      HttpStatus.CONFLICT
    );
  }
  
  static workspaceConflict() {
    return new ApiException(
      'WORKSPACE_CONFLICT',
      'Workspace was modified by another run, please retry',
      HttpStatus.CONFLICT
    );
  }
  
  static validation(errors: Array<{ field: string; message: string }>) {
    return new ApiException(
      'VALIDATION_ERROR',
      'Invalid request body',
      HttpStatus.BAD_REQUEST,
      errors
    );
  }
}

// 使用
if (!task) {
  throw ErrorFactory.taskNotFound(taskId);
}
```

### 日志中间件

```typescript
@Injectable()
export class ErrorLoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(
      catchError((error) => {
        const request = context.switchToHttp().getRequest();
        const authContext = request.authContext;
        
        // 记录错误日志
        logger.error({
          code: error.code || 'INTERNAL_ERROR',
          message: error.message,
          context: {
            taskId: request.params.taskId,
            runId: request.params.runId,
            userId: authContext?.userId,
            tenantId: authContext?.tenantId,
          },
          stack: error.stack,
          request: {
            method: request.method,
            path: request.path,
            ip: request.ip,
            userAgent: request.headers['user-agent'],
          },
        });
        
        return throwError(() => error);
      })
    );
  }
}
```

## 常见问题

### Q1: 如何处理多语言错误提示？

在 `message` 字段保持英文，前端根据 `code` 映射到本地化文案：

```typescript
const ERROR_MESSAGES_ZH: Record<string, string> = {
  AUTH_REQUIRED: '请先登录',
  // ...
};

const ERROR_MESSAGES_EN: Record<string, string> = {
  AUTH_REQUIRED: 'Please login to continue',
  // ...
};
```

### Q2: 如何处理敏感信息泄露？

生产环境不返回 `stack`，只返回 `code` 和用户友好的 `message`：

```typescript
if (process.env.NODE_ENV === 'production') {
  delete errorResponse.stack;
  delete errorResponse.internalMessage;
}
```

### Q3: 如何测试错误处理？

编写集成测试：

```typescript
describe('Error Handling', () => {
  it('should return AUTH_REQUIRED when not logged in', async () => {
    const response = await request(app.getHttpServer())
      .post('/api/task')
      .send({ title: 'Test' })
      .expect(401);
    
    expect(response.body.code).toBe('AUTH_REQUIRED');
  });
  
  it('should return TASK_NOT_FOUND for invalid taskId', async () => {
    const response = await request(app.getHttpServer())
      .get('/api/task/invalid_id')
      .set('Cookie', sessionCookie)
      .expect(404);
    
    expect(response.body.code).toBe('TASK_NOT_FOUND');
  });
});
```
