# 历史记录与上下文优化
每轮无需将完整历史注入 Pi。

推荐上下文：

```text
task summary
  + recent N messages
  + latest artifact manifest
  + current user prompt
```

## Task Summary 刷新策略

**🚨 对齐 STAGE0_SPEC §10.3 / §5.7**：阶段 0 的 summary refresh 是 **system background job**，每个 run 完成后触发一次**异步**刷新，**不阻塞用户**，失败只写 warning 和 failure counter，**不自动切换** task 到 `running_degraded`。

```typescript
interface TaskSummaryPolicy {
  // 触发条件（满足任一即触发，但永远是异步后台执行，不阻塞下一个 run）
  refreshConditions: {
    runsCompleted: 10;      // 累计完成 10 个 run 后刷新
    estimatedTokens: 30000; // 估算 context 超过 30k tokens
    artifactFilesChanged: 50; // artifact 文件数变化超过 50
  };

  // 失败处理（不降级 task 状态）
  failurePolicy: {
    maxConsecutiveFailures: 3; // 超过 3 次写 warning；阶段 0 不切换 task 状态
    onError: "increment_counter_and_warn"; // 保留 last_successful summary
  };
}

class TaskSummaryManager {
  // maybeRefreshSummary 由 run 完成后异步调用；永不阻塞调用者
  async maybeRefreshSummary(taskId: string, completedRunId: string): Promise<void> {
    const task = await taskStore.getTask(taskId);
    const summary = await taskStore.getTaskSummary(taskId);

    const runsSinceLastSummary = task.completedRuns - (summary?.updatedAtRunCount ?? 0);
    const estimatedTokens = this.estimateContextTokens(task, summary);
    const artifactDelta = this.calculateArtifactDelta(task, summary);

    const shouldRefresh =
      runsSinceLastSummary >= 10 ||
      estimatedTokens >= 30000 ||
      artifactDelta.filesChanged >= 50;

    if (!shouldRefresh) return;

    // 🚨 异步执行，不等待，不阻塞 run / message 流程
    void this.refreshSummaryAsync(taskId, completedRunId).catch((err) => {
      logger.warn(`Background summary refresh failed for task ${taskId}`, err);
    });
  }

  // 单一异步刷新入口；阶段 0 没有 sync/force 路径
  private async refreshSummaryAsync(taskId: string, completedRunId: string): Promise<void> {
    try {
      const input = await getSummaryRefreshInput(taskId, completedRunId); // 见 STAGE0_SPEC §10.3.1
      const newSummary = await this.generateSummary(input);

      await taskStore.updateTaskSummary({
        taskId,
        summary: newSummary,               // ✅ 统一字段名 summary（STAGE0_SPEC §3.3）
        updatedRunId: completedRunId,
        consecutiveFailures: 0,
        lastSuccessfulRunId: completedRunId,
      });
    } catch (error) {
      // 🚨 失败只递增计数 + 写 warning；不切换 task 到 running_degraded
      const prev = await taskStore.getTaskSummary(taskId);
      const consecutiveFailures = (prev?.consecutiveFailures ?? 0) + 1;

      await taskStore.updateTaskSummary({
        taskId,
        consecutiveFailures,
      });

      await taskStore.appendWarning(taskId, `summary_refresh_failed (${consecutiveFailures})`);
    }
  }

  private async generateSummary(input: SummaryRefreshInput): Promise<string> {
    const prompt = this.buildSummaryPrompt(input);
    return this.llmClient.generate({
      model: "deepseek-v4-flash",
      prompt,
      maxTokens: 2000,
    });
  }

  private estimateContextTokens(task: Task, summary: TaskSummary | null): number {
    const summaryTokens = summary ? summary.summary.length / 4 : 0;
    const messagesTokens = task.recentMessagesLength / 4;
    return summaryTokens + messagesTokens;
  }

  private calculateArtifactDelta(task: Task, summary: TaskSummary | null): { filesChanged: number } {
    return { filesChanged: Math.abs((task.artifactCount ?? 0) - (summary?.artifactCount ?? 0)) };
  }
}
```

### 触发条件（参考值，可调）

| 条件 | 阈值 | 说明 |
|------|------|------|
| runsCompleted | ≥ 10 | 累计完成的 run 数 |
| estimatedTokens | ≥ 30000 | context 估算 token 数 |
| artifactFilesChanged | ≥ 50 | 自上次 summary 以来 artifact 变化 |

> 阈值只是参考；STAGE0_SPEC §10.3 要求每个 run 完成后都触发一次异步 refresh，由 manager 决定是否真正执行。

## 失败处理

阶段 0 失败处理（STAGE0_SPEC §5.7 / §10.3）：

- 递增 `consecutive_failures`，写 warning。
- **不自动切换** task 到 `running_degraded`；task 状态保持 `running`。
- 下次 run 构建上下文时使用 `last_successful` summary；无则使用空 summary。
- 不影响用户继续发消息，不影响 run 完成。

## TaskSummary 字段（对齐 STAGE0_SPEC §10.4 / schema.sql）

```typescript
interface TaskSummary {
  taskId: string;
  summary: string;              // ✅ 统一为 summary（不再使用 content）
  runCount: number;
  tokenCount: number;
  artifactCount: number;
  updatedAtRunCount: number;    // summary 更新时的 task.completedRuns
  consecutiveFailures: number;
  lastSuccessfulRunId: string | null;
  createdAt: number;
  updatedAt: number;
}
```

## 关键改进

- **永不阻塞**：所有 summary refresh 都是 system background job，不阻塞 run / message 流程。
- **失败不降级**：失败只写 warning 和 counter，不切换 task 到 `running_degraded`（阶段 0 保留枚举但不自动用）。
- **字段统一**：`summary` 而非 `content`；接口字段对齐 schema.sql 和 STAGE0_SPEC §10.4。
- **输入边界**：刷新输入遵循 STAGE0_SPEC §10.3.1 的窗口策略，避免长任务 OOM。
